Chrome Cardboard WebXR neck-model bug

On Android Chrome's Google Cardboard immersive-vr path, the viewer position from XRFrame.getViewerPose() is wrong. The Cardboard SDK runs its 3-DoF neck model on the inverse of the head orientation. Chrome conjugates the orientation — which corrects it — but "fixes" the position by negating the SDK's vector, and negating a neck model built from the inverse rotation can't recover the right position. Orientation is correct; only position is affected. See the write-up for the analysis and the exact upstream location.
